# Практическая работа №4: Многопоточность, фоновые задачи и службы

## Тема
Изучение механизмов многопоточности, фоновых задач и служб (`Service`) для выполнения длительных операций без блокировки основного потока (UI-потока).

## Обзор
Четвертая практическая работа посвящена критически важной теме в Android-разработке — обеспечению отзывчивости пользовательского интерфейса путем вынесения ресурсоемких операций в фоновые потоки.

## Содержание модулей

| Модуль | Описание | Ключевые концепции |
| :--- | :--- | :--- |
| **cryptoloader** | Пример использования `Loader` (или его современного аналога `AsyncTaskLoader`) для асинхронной загрузки данных. В данном случае, вероятно, для выполнения криптографических операций. | `Loader` API, асинхронная загрузка данных, взаимодействие с UI-потоком через `LoaderManager`. |
| **data_thread** | Базовый пример работы с потоками данных. Демонстрация создания и запуска нового потока (`Thread`) для выполнения задачи, а также потенциальные проблемы синхронизации. | Класс `Thread`, метод `run()`, основы многопоточности в Java. |
| **looper** | Пример использования `Looper` и `Handler` для организации очереди сообщений в потоке. Это позволяет потоку выполнять задачи последовательно и взаимодействовать с другими потоками. | `Looper`, `Handler`, `Message`, организация цикла обработки сообщений. |
| **serviceapp** | Примеры использования `Service` для выполнения длительных операций в фоне, которые не требуют UI. Включает `Started Service` и, возможно, `Bound Service`. | Жизненный цикл `Service`, запуск и остановка службы, выполнение фоновых задач. |
| **thread** | Дополнительный пример работы с классом `Thread`, возможно, с более сложными сценариями, такими как использование `Runnable`. | Интерфейс `Runnable`, передача задач в поток. |
| **workmanager** | Пример использования `WorkManager` — рекомендуемого Google API для отложенного, гарантированного выполнения фоновых задач. | `WorkManager`, `Worker`, `OneTimeWorkRequest`, `PeriodicWorkRequest`, ограничения (`Constraints`). |

## Ключевые навыки, освоенные в работе
1.  **Управление потоками:** Создание и управление фоновыми потоками для предотвращения ANR (Application Not Responding).
2.  **Фоновые компоненты:** Использование `Service` и `WorkManager` для выполнения задач, которые должны продолжаться даже при отсутствии активного UI.
3.  **Синхронизация:** Понимание необходимости синхронизации и безопасного взаимодействия между фоновыми и UI-потоками.
