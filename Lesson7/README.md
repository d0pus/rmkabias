# Практическая работа №7: Сетевое взаимодействие и сторонние библиотеки

## Тема
Реализация сетевого взаимодействия (HTTP, сокеты) и интеграция сторонних сервисов и библиотек (например, Firebase).

## Обзор
Седьмая практическая работа посвящена подключению приложения к внешнему миру. Рассматриваются как низкоуровневые механизмы (сокеты), так и высокоуровневые API (HTTP-запросы, облачные сервисы).

## Содержание модулей

| Модуль | Описание | Ключевые концепции |
| :--- | :--- | :--- |
| **firebaseauth** | Пример использования сторонней библиотеки для аутентификации через Firebase. Демонстрирует регистрацию, вход, выход и верификацию пользователя. | Firebase Authentication, интеграция SDK, работа с облачными сервисами, обработка состояний аутентификации. |
| **httpurlconnection** | Приложение для выполнения HTTP-запросов и получения данных. Используется для получения внешнего IP-адреса, а также, возможно, данных о погоде с публичных API. | `HttpURLConnection`, выполнение сетевых запросов в фоновом потоке, парсинг JSON/XML ответов. |
| **timeservice** | Пример сетевого взаимодействия с использованием сокетов для получения времени с удаленного сервера (например, NTP-сервера). | Сокеты (`Socket`), низкоуровневое сетевое программирование, работа с потоками ввода/вывода. |

## Ключевые навыки, освоенные в работе
1.  **Сетевое программирование:** Реализация как низкоуровневого (сокеты), так и высокоуровневого (HTTP) сетевого взаимодействия.
2.  **Интеграция API:** Подключение к внешним API и облачным сервисам (Firebase).
3.  **Асинхронность:** Выполнение всех сетевых операций в фоновых потоках для сохранения отзывчивости UI.

*   **Язык программирования:** Java/Kotlin.
*   **Среда разработки:** Android Studio.
*   **Целевая платформа:** Android SDK.
